はじめに

Movable Type theme "manual"とは

「manual」は、Movable Type専用の、マニュアル作成のためのテーマです。HTMLでA4印刷に最適化されたマニュアルを作成し、PDFもしくは紙で出力することができます。
CSS等は、基本的にMacOS＋Safari＋Acrobat前提となっています。

GitHubでリポジトリを公開しています。

https://github.com/webbingstudio/movabletype_theme_manual

　

テーマ構成

ウェブサイト用テーマと、マニュアル別のブログテーマの親子構成となっています。両方ともインストールしてください。ヘッダ・フッタ・bootstrap.cssは親テーマに含まれています。

なぜマニュアルをMTで作成するのか

マニュアル作成の手段

CMS構築の案件では、各サイト独自の操作マニュアルを作成する必要があります。マニュアルの作成には様々な方法があります。

  • Word・Excel等のOffice系ソフト
  • Office系ソフトからPDF変換
  • Googleドライブ・Microsoft OneDrive等のオンライン共有
  • マニュアル作成専用のアプリケーション
  • PDFからIllustrator等の描画ソフト
  • プロジェクト管理ツールのWiki

マニュアル共有に関わる問題

さまざまな方法を試したのですが、以下の問題があり、なかなか良い方法が見つかりませんでした。

  • Office系ソフトはクライアントのOSやバージョンによってページが崩れる
  • Wikiは体裁の自由度が低く、様々なサイズのスクリーンショットを配置しにくい
  • オンライン共有はクライアントのリテラシが低いと見落とされやすい
  • Officeやオンラインツールを使用すると、スクリーンショットが縮小され調整がきかない
  • 専用アプリケーションは諸経費が大きく、乗り換えがしづらい
  • 簡単な体裁で良い、と言われると既存フォーマットが使えず、かえって面倒

また、一般的でないツールを使っていると、中間クライアント側での改版・加筆の要望があった場合、常に書いた本人が修正しなければならないのも悩みの種でした。

たどり着いた結論（暫定）

最近は、ブラウザのプリント機能の精度が上がったため、Movable Typeを通してHTMLで作成し、PDFで出力する方法を取っています。
（今ご覧になっているページです）

CMS＋bootstrapをベースとしているため修正がしやすく、画像をCSSで縮小しているため自由にサイズ調整ができます。

また、Movable TypeはHTMLを静的に出力できるため、画像パスを相対にしておくと、ローカル環境でも閲覧可能な状態でアーカイブできます。


このたび、一年ほど実際に使用しているMovable Typeテーマを、一般向けに公開しました。

Movable Typeを利用していて、マニュアル作成にお悩みの方は参考になれば幸いです。
